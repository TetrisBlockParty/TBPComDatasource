# TBPComDatasource

## Overview

This repository serves as a **datasource** for storing tournament participation information. It maintains historical records and current standings for group tournaments, providing a structured data storage solution for tournament tracking and analysis.

## Purpose

The repository stores and tracks:
- **Tournament participation data** - Records of participants and their performance
- **Tournament standings** - Current and historical rankings
- **Participant details** - Individual participant scores and statistics
- **Time-series data** - Snapshots of tournament states at different timestamps

## Repository Structure

```
grouptmt/
└── [TOURNAMENT_ID]/
    ├── [DATE]/
    │   └── [TIMESTAMP]/
    │       ├── standings.md
    │       └── details/
    │           └── [participant].md
    └── latest/
        ├── standings.md
        └── details/
            └── [participant].md
```

### Directory Organization

- **`grouptmt/`** - Root directory for group tournament data
- **`[TOURNAMENT_ID]/`** - Tournament identifier (e.g., `NL2025`)
- **`[DATE]/`** - Date folder in `YYYY-MM-DD` format
- **`[TIMESTAMP]/`** - Specific timestamp snapshots in `YYYY-MM-DD_HH-MM-SS` format
- **`latest/`** - Symlink or copy of the most recent tournament state
- **`standings.md`** - Overall tournament standings table
- **`details/[participant].md`** - Individual participant performance details

## Data Format

### Standings File
Contains a markdown table with:
- Rank
- Score
- Participant name
- Link to detailed participant information
- Timestamp of data generation

### Participant Details File
Contains:
- Total number of participants
- Individual participant scores
- Ranking information
- Timestamp of data generation

## Usage

This datasource is designed to be:
- **Version-controlled** - Track changes over time using Git
- **Human-readable** - Markdown format for easy viewing
- **Machine-parsable** - Structured format for automated processing
- **Historical** - Maintains snapshots at different timestamps

## Data Generation

All data files are automatically generated by the tournaments datasource generation tool, which periodically captures tournament states and stores them in this repository.
